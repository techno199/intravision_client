Если вкратце, то на фронте есть несколько компонент, они все подвязаны непосредственно  на сервисы, а не друг на друга. 
Сервисы отвечают за свой класс функционала: сервис для монет, для напитков, для авторизации и для уведомлений. 
Каждое действие сопровождается вызовом по API для синхронизации данных, кроме подсчета текущего баланса 
(он считается локально, при перезапуске "автомата" сбрасывается). Монеты: есть разные типы монет для каждого отдельного автомата
(хотя автомат всего один в данном случае).Так же и с напитками - у каждого автомата свой набор. Напитки можно 
добавлять, удалять, редактировать.
Авторизация через куки. Методы, требующие авториации, обвешены атрибутом [Authorize].
На сервере тоже есть разделение на Сервисы и Контроллеры.
Есть один сервис для общения с бд (я туда все методы накидал неависимо от тематики),
есть разные типовые контроллеры: для монеток, напитков и авторизации.
Для создания модели использовал миграции (code first)
-----
Доп пункты: 
1) Выбор нескольких напритков перед выдачей сдачи.
2) Возможность работать с любым номиналом (Возможность добавлять монеты, также как и напитки (свой отдельный CRUD)).
